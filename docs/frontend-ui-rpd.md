# thaven 前端 UI 设计需求说明

本文档面向 UI 设计。目标是说明 thaven 每个页面需要设计什么、保留哪些现有功能、需要覆盖哪些交互状态。

设计时只关注前端界面和用户体验。Rust 后端、接口、数据存储、技术实现不需要设计处理。

## 1. 产品定位

thaven 是一款桌面端 Wallhaven 壁纸工具，核心用途是：

- 浏览在线壁纸
- 筛选和搜索壁纸
- 预览大图
- 下载壁纸
- 设置系统壁纸
- 管理本地已下载壁纸
- 查看下载任务
- 配置 API Key 和下载目录

整体设计应偏向极简工具型桌面应用，重点是清晰、高效、图片展示舒服。视觉上可以有轻科技感和优雅的交互动效，但不需要做成营销页面、复杂内容站或重装饰应用。

## 2. 全局设计要求

### 2.1 应用整体结构

应用需要包含一个固定主导航，当前有 5 个主页面：

- 在线壁纸
- 本地壁纸
- 下载中心
- 设置
- 关于

推荐结构：

- 左侧为主导航栏。
- 右侧为页面内容区。
- 页面内容区顶部可有页面标题、状态摘要和主要操作。
- 壁纸图片应是视觉主体，不要让导航和控件过度抢占空间。

### 2.2 设计风格

整体风格建议：

- 极简：界面结构简单，信息层级清楚，减少不必要装饰。
- 工具感：所有元素服务于浏览、筛选、下载、设置壁纸，不做无功能的视觉堆叠。
- 轻科技：可以使用细线、半透明面板、柔和高亮、轻微发光或玻璃质感，但要克制。
- 优雅交互：按钮、卡片、弹窗、筛选面板需要有顺滑、轻量的过渡反馈。
- 图片优先：壁纸是主角，界面控件要低干扰。
- 桌面端精致感：布局紧凑但不拥挤，适合长时间使用。
- 支持暗色模式：暗色模式应自然、干净，不要过黑或过紫。

设计关键词：

- 极简
- 轻科技
- 优雅交互
- 图片优先
- 紧凑清晰
- 操作明确
- 克制高级

避免：

- 大面积营销式首屏
- 过大的标题和空白
- 过度圆角、厚重阴影
- 控件堆叠过多导致壁纸区域被压缩
- 只有单一色系的界面
- 过强渐变和大面积发光
- 复杂背景纹理
- 过度拟物化

### 2.3 交互风格

交互需要体现“轻、快、顺”：

- 悬停反馈要轻微明确，例如卡片轻微上浮、图片轻微放大、操作按钮浮现。
- 点击反馈要迅速，不要使用过长动画。
- 页面切换可以有轻微淡入，但不要影响操作效率。
- 弹窗出现应顺滑，背景弱化但不应完全打断视觉上下文。
- 筛选面板展开/收起需要自然，不要突然跳变。
- 加载状态尽量使用骨架、轻量进度或柔和闪烁，不要使用夸张动画。
- 成功和错误反馈要短暂、清楚，不要打断用户连续浏览。

动效建议：

- 卡片 hover：120ms-180ms。
- 按钮 hover/click：80ms-140ms。
- 弹窗打开：160ms-220ms。
- 筛选面板展开：140ms-200ms。
- 图片加载过渡：淡入即可。

### 2.4 色彩和质感

整体色彩应克制，避免花哨。建议：

- 主背景使用干净的浅灰、深灰或接近黑的中性色。
- 主色可以选择蓝、青、蓝紫等偏科技感的颜色，但只作为强调色。
- 重要操作使用主色，危险操作使用红色，成功状态使用绿色。
- 面板可以使用轻微透明、细边框、微弱阴影，但不要做厚重卡片。
- 暗色模式下可以使用轻微冷色调，让界面有科技感，但不要整体变成单一蓝紫色。

壁纸卡片上的 UI 应保持低透明度和低占用，让图片内容最大化展示。

### 2.5 通用状态

所有页面都需要考虑这些状态：

- 加载中
- 空数据
- 加载失败
- 操作成功
- 操作失败
- 禁用状态
- 长文本溢出
- 小窗口宽度下的布局

图片相关状态：

- 图片加载中
- 图片加载失败
- 缩略图占位
- 大图预览加载中

## 3. 主导航设计

### 3.1 需要设计的内容

主导航包含：

- 应用名称或标识：thaven
- 在线壁纸入口
- 本地壁纸入口
- 下载中心入口
- 设置入口
- 关于入口
- 当前页面高亮状态

### 3.2 交互要求

- 点击导航项切换页面。
- 当前页面需要有明显选中状态。
- 导航可以支持收起/展开。
- 收起状态下仍能通过图标识别页面。
- 图标按钮需要有悬停提示或可理解的文字。

### 3.3 设计注意

- 导航不应太宽，避免压缩壁纸展示区。
- 在线壁纸是最高频页面，应放在最上方。
- 下载中心如存在下载中任务，可以考虑显示轻量状态提示，例如小圆点或数字。

## 4. 在线壁纸页

在线壁纸页是核心页面，也是设计优先级最高的页面。

### 4.1 页面目标

用户进入应用后，可以立即浏览在线壁纸，并通过搜索和筛选快速找到想要的图片。

### 4.2 页面需要设计的区域

在线壁纸页需要包含：

- 页面标题或当前位置标识
- 搜索输入框
- 分类筛选
- 纯度筛选
- 分辨率筛选
- 比例筛选
- 颜色筛选
- 排序方式
- 升序/降序
- 排行时间范围
- 随机 seed 输入
- 搜索/刷新按钮
- API Key 缺失提示
- 结果统计信息
- 壁纸网格
- 加载更多状态
- 空状态
- 错误重试状态
- 壁纸预览弹窗

### 4.3 搜索和筛选区

需要保留的筛选项：

- 关键词
- 分类：综合、动漫、人物
- 纯度：SFW、Sketchy、NSFW
- 分辨率：不限、至少 1280x720、至少 1600x900、至少 1920x1080、至少 2560x1440、至少 3840x2160、精确 1280x720、1600x900、1920x1080、2560x1440、3840x2160、3440x1440
- 比例：不限、横屏、竖屏、16x9、16x10、21x9、32x9、4x3、1x1
- 颜色：红、粉、紫、蓝、青、绿、黄、橙、黑、白
- 排序：最新、相关、随机、浏览、收藏、排行
- 顺序：降序、升序
- 排行时间：1 天、3 天、1 周、1 月、3 月、6 月、1 年
- 随机 seed

显示规则：

- 默认显示最新 SFW 壁纸。
- 未设置 API Key 时，NSFW 需要禁用。
- 未设置 API Key 时，页面需要提示“未设置 API Key，NSFW 已禁用”。
- 只有选择“排行”排序时，显示排行时间范围。
- 只有选择“随机”排序时，显示 seed 输入。
- 搜索栏内容较多，设计时可以把低频筛选收进高级筛选面板。

### 4.4 壁纸网格

每张在线壁纸卡片需要展示：

- 缩略图
- 分辨率
- 文件大小
- 文件格式
- 分类
- 纯度
- 预览操作
- 下载操作
- 设置为壁纸操作

卡片设计建议：

- 缩略图占卡片主要面积。
- 卡片信息保持轻量，不要遮挡太多图片。
- 操作按钮可以默认弱化，悬停时增强。
- 下载中或设置壁纸中，需要显示操作中的状态。
- 缩略图加载失败时，需要显示清晰占位。

### 4.5 结果信息

页面需要显示：

- 总壁纸数量
- 当前页
- 总页数
- 随机模式下的 seed

这些信息可以做得轻量，不应比壁纸内容更抢眼。

### 4.6 加载更多

当前功能是向下滚动自动加载更多。

需要设计：

- 首屏加载状态
- 底部加载更多状态
- “继续向下滚动加载更多”
- “已加载全部”

### 4.7 空状态

没有搜索结果时，需要设计空状态：

- 文案：没有找到壁纸
- 保留搜索筛选区，方便用户调整条件
- 可提供“重置筛选”操作，但不是必须

### 4.8 错误状态

加载失败时，需要设计：

- 错误标题：加载失败
- 错误原因展示区域
- 重试按钮

### 4.9 在线壁纸预览弹窗

点击缩略图或预览后打开。

预览弹窗需要包含：

- 大图展示区域
- 大图加载状态
- 图片信息层
- 下载当前壁纸按钮
- 设置当前壁纸按钮
- 关闭入口

图片信息需要展示：

- 分辨率
- 文件大小
- 文件类型
- 分类
- 纯度
- 标签列表

设计注意：

- 大图不能被拉伸变形。
- 信息层不能遮挡主要画面太多。
- 信息层可以悬浮显示，也可以放在侧边或底部。
- 小窗口下信息层需要可滚动或折叠。

## 5. 本地壁纸页

### 5.1 页面目标

用户可以查看下载目录中的本地壁纸，并进行预览、设置壁纸、显示位置和删除操作。

### 5.2 页面需要设计的区域

本地壁纸页需要包含：

- 页面标题
- 当前目录输入/展示
- 扫描按钮
- 打开目录按钮
- 本地壁纸总数
- 本地壁纸网格
- 分页
- 空状态
- 加载状态
- 本地预览弹窗
- 删除确认

### 5.3 目录工具区

需要展示：

- 当前下载目录
- 扫描
- 打开目录

设计注意：

- 路径可能很长，需要支持省略或横向滚动。
- 扫描时按钮需要 loading 状态。
- 目录区域应该是工具条，不要占太高空间。

### 5.4 本地壁纸卡片

每张本地壁纸卡片需要展示：

- 缩略图
- 文件名
- 图片尺寸
- 文件大小
- 修改时间
- 预览按钮
- 设置为壁纸按钮
- 显示位置按钮
- 删除按钮

设计注意：

- 文件名可能很长，需要省略显示，悬停或预览中可看到完整文件名。
- 删除操作需要用危险色表达。
- 缩略图加载中和加载失败都需要占位。

### 5.5 分页

本地壁纸使用分页，不是无限滚动。

需要设计：

- 当前页
- 总页数
- 上一页/下一页
- 页码切换

如果只有一页，不需要显示分页。

### 5.6 本地预览弹窗

本地预览需要展示：

- 本地图片大图
- 文件名
- 图片尺寸
- 文件大小
- 创建时间
- 设置为壁纸
- 显示位置
- 删除

设计注意：

- 文件名和路径相关信息可能很长。
- 删除按钮需要明显区别于普通操作。
- 删除后弹窗会关闭，列表刷新。

### 5.7 删除确认

删除本地壁纸前必须确认。

确认框需要展示：

- 标题：删除本地壁纸
- 文案：确认删除该文件
- 文件名
- 删除按钮
- 取消按钮

删除按钮需要使用危险色。

## 6. 下载中心页

### 6.1 页面目标

用户可以查看所有下载任务，了解下载状态，处理失败任务，并打开已完成文件的位置。

### 6.2 页面需要设计的区域

下载中心页需要包含：

- 页面标题
- 下载任务统计
- 清理已结束按钮
- 下载任务列表
- 空状态

### 6.3 任务统计

需要展示：

- 下载中数量
- 已完成数量
- 失败数量

统计信息建议放在页面顶部，轻量展示。

### 6.4 下载任务列表

每条任务需要展示：

- 缩略图
- 文件名
- 状态标签
- 已存在标签
- 开始时间
- 结束时间
- 进度条
- 已下载大小 / 总大小
- 本地路径或目标目录
- 错误信息
- 打开位置按钮
- 重试按钮
- 移除按钮

状态包括：

- 下载中
- 已完成
- 失败

按钮显示规则：

- 已完成任务：可打开位置，可移除。
- 失败任务：可重试，可移除。
- 下载中任务：显示进度，不允许移除。
- 未下载完成时，打开位置不可用。

### 6.5 空状态

没有下载记录时展示：

- 文案：暂无下载记录
- 可引导用户去在线壁纸页下载，但不是必须

### 6.6 设计注意

- 下载中心适合列表，不建议做图片卡片网格。
- 失败信息需要可读，不要只显示红色图标。
- 本地路径可能很长，需要省略。
- 当前功能不支持真实暂停、恢复、取消，不要设计这些按钮。
- 当前功能不统计实时下载速度，不要设计下载速度字段。

## 7. 设置页

### 7.1 页面目标

用户可以配置 API Key、下载目录，并重置应用数据。

### 7.2 页面需要设计的区域

设置页需要包含 3 个分组：

- API Key
- 下载目录
- 应用数据

### 7.3 API Key 分组

需要包含：

- API Key 输入框
- 保存 API Key 按钮
- 简短说明：设置后可访问账号允许的内容，NSFW 依赖 API Key

设计注意：

- API Key 输入框应按密码或隐藏内容处理。
- 保存成功需要有反馈。
- 输入为空时也允许保存，表示清空 API Key。

### 7.4 下载目录分组

需要包含：

- 下载目录输入框
- 选择目录按钮
- 打开目录按钮
- 保存下载目录按钮

设计注意：

- 路径可能很长。
- 用户可以手动输入，也可以选择目录。
- 打开目录和保存目录都需要失败反馈。

### 7.5 应用数据分组

需要包含：

- 重置应用数据按钮
- 风险说明

重置会清空：

- API Key
- 下载目录
- 下载记录

重置不会删除：

- 已下载到本地的图片文件

重置前必须确认。

## 8. 关于页

### 8.1 页面目标

展示应用基础信息和相关链接。

### 8.2 页面需要设计的内容

关于页需要包含：

- 应用名称：thaven
- 应用说明：基于 Tauri 的 Wallhaven 壁纸工具，用于浏览在线壁纸、下载图片，并管理本地壁纸
- 当前版本
- 许可证：MIT License
- 项目仓库链接
- Wallhaven API 文档链接

### 8.3 设计注意

- 关于页保持简洁。
- 不需要大面积宣传图。
- 外部链接需要看起来可点击。
- 打开失败时需要错误反馈。

## 9. 404 页面

### 9.1 页面目标

当用户进入不存在的页面时，给出明确提示并允许返回可用页面。

### 9.2 页面需要设计的内容

404 页面需要包含：

- 页面不存在提示
- 返回在线壁纸按钮
- 返回上一页按钮，可选

## 10. 弹窗和反馈

### 10.1 需要设计的弹窗

- 在线壁纸预览
- 本地壁纸预览
- 删除本地壁纸确认
- 重置应用数据确认

### 10.2 需要设计的消息反馈

成功反馈：

- API Key 已保存
- 下载目录已保存
- 已下载到指定路径
- 已设置为壁纸
- 已删除
- 应用数据已重置

警告反馈：

- 已下载过
- 文件尚未下载完成
- 未设置 API Key，NSFW 已禁用

错误反馈：

- 搜索失败
- 下载失败
- 设置壁纸失败
- 扫描本地壁纸失败
- 打开目录失败
- 删除失败
- 打开外部链接失败

## 11. 小窗口适配

设计需要覆盖较窄窗口，尤其是：

- 主导航收起或变窄。
- 搜索筛选区允许换行、横向滚动或折叠。
- 壁纸卡片网格列数自动减少。
- 下载任务列表在窄窗口下可以纵向堆叠。
- 长文件名、长路径不能撑破布局。
- 弹窗不能超出窗口，需要内部滚动。

## 12. 第一版设计交付建议

第一版设计建议至少交付这些画面：

- 主布局 + 侧边导航
- 在线壁纸页默认状态
- 在线壁纸页筛选展开状态
- 在线壁纸页加载中、空状态、错误状态
- 在线壁纸卡片 hover 状态
- 在线壁纸预览弹窗
- 本地壁纸页默认状态
- 本地壁纸预览弹窗
- 删除确认框
- 下载中心有任务状态
- 下载中心空状态
- 设置页
- 重置确认框
- 关于页
- 404 页面

如果时间有限，优先级如下：

1. 在线壁纸页
2. 在线壁纸预览弹窗
3. 本地壁纸页
4. 下载中心页
5. 设置页
6. 主导航
7. 关于页和 404

## 13. 设计验收清单

设计稿需要确认：

- 现有 5 个主页面都覆盖。
- 在线壁纸页保留全部筛选条件。
- 在线壁纸卡片保留预览、下载、设置壁纸。
